# 热点功能

看到最近 bot 主动帖子都是一些所感所想，流量都是跟着热度走的，必须发帖跟随热度。
bot 自动跟随热点，无需人工审核每一步。


## 一、热度发现

### 关注方向（按优先级排序）

- 【最高优先级】AI + vibe coding：AI 编程工具、AI 工作流自动化、AI agent、LLM 辅助开发、cursor/windsurf/copilot 等工具生态、低代码与 AI 结合
- 【高优先级】AI 工作流：AI 如何改变日常工作方式、AI 自动化实践、人与 AI 协作模式
- 【中等优先级】创业/startup、产品/增长、开发者工具、web3/加密货币、金融/金融科技、半导体/光模块/硬件、自媒体创作
- 【低优先级】AI 其他方面（模型发布、学术论文、融资新闻）——仅供评分参考，不优先选取

### 热度来源

- Hacker News（news.ycombinator.com）—— AI/技术圈风向标，高票帖代表当前讨论热点
- X Trending / For You 时间线（后续接入）——最直接的热度信号
- 其他来源（后续扩展）：微信公众号热文、知乎热榜、HuggingFace Papers

### 发现机制
- 定时从 HN API 拉取 top 30 热门
- LLM 逐条筛选评分：是否与关注方向相关，按优先级加权打分
- 相关度 >= 3 的热点自动加入发帖队列
- SQLite 去重：同一热点只发一次


## 二、跟随热点

### 核心原则
单纯转发不行，必须有解析、有观点、有立场。

### 发帖模式

1. **观点型**：「XX 发生了，我的看法是 YY，不同意 ZZ 因为……」
2. **解读型**：「XX 发布了什么，翻译成人话就是……，对行业意味着……」
3. **反共识型**：「所有人都在说 XX，但我觉得他们漏了一个关键点……」
4. **关联型**：「XX 这件事，让我想起之前 YY，本质上是同一个问题……」

### 生成要求
- LLM prompt 必须要求输出观点，不是摘要
- 不要用 hashtag，不要 emoji，禁止用破折号（——），禁止用引号
- 赋予模型明确的 persona 约束，必须有立场
- 每条帖子包含：热点引用 → 观点声明 → 论据支撑 → 结论/号召
- 热点以 news_react 类型流入 post_topics.json 队列，复用现有 post_once.py 管线


## 三、反馈机制

内容不仅是给 X 上的粉丝看的，也是做给自己看的，所以我的反馈最重要。

### 需要提供的反馈维度

1. **方向对与否**：这个话题方向对不对？不想让 bot 碰的领域要明确标出
2. **观点像不像你**：bot 抛的观点符合你的立场吗？哪些地方"这不是我会说的话"
3. **语气调性**：太激进？太保守？太学术？太油滑？
4. **好样例 vs 坏样例**：你自己发的帖子里哪些你觉得"这就是我想要 bot 达到的水平"——给具体链接，比任何 prompt 调优都有效

### 反馈通道
- Telegram 命令直接对已发帖子打分/评价（/rate、/review）
- 反馈数据存入 state/，作为后续 prompt 优化的参考样本


## 四、实现摘要

### 新增文件
- `src/hotspot/store.py` — SQLite 热点存储（去重、查询、统计）
- `src/hotspot/discover.py` — HN API 抓取 + LLM 过滤评分
- `discover_hotspots.py` — CLI 入口脚本

### 修改文件
- `common.py` — 新增热点状态路径常量
- `bot_daemon.py` — 新增热点 job 调度（默认每 2 小时）、carry_over、Telegram 命令
- `sync_tg_commands.py` — 新增 /hotspot_discover、/hotspot_status
- `.env.example` — 新增热点环境变量

### 环境变量
- `X_HOTSPOT_ENABLED` — 开关（默认 1）
- `X_HOTSPOT_INTERVAL_SECONDS` — 发现间隔（默认 7200）
- `X_HOTSPOT_DAILY_LIMIT` — 每日热点帖上限（默认 3）
- `X_HOTSPOT_GUARD_SECONDS` — 保护间隔（默认 600）
